## Task
Debug and fix audio overlap/chime cutoff after match events

## Root Cause Analysis
Audio playback was triggered per-tile in `Tile.MoveTile()`. During batched dequeue animation dispatches (5–15 tiles per frame), the single `AudioStreamPlayer` (default `max_polyphony=1`) received rapid `.Play()` calls that cut off previous audio, causing overlap and eventual exhaustion where chimes stopped playing.

## Solution Implemented

### AudioStreamPlayer Event Consolidation
- Moved sound triggering from `Tile.MoveTile()` to `GameBoard._Process()`
- Implemented `_lastSoundRound` guard to fire sounds once per animation round
- Removed all per-tile audio calls from `Tile.cs`

### Off-by-One Fix
- Fixed array access where rounds 4+ attempted to play non-existent `Sound_MatchHypeLevel5` node
- Capped hype level at `Globals.MaxHypeLevel - 1`

## Changes Made

**File:** `Scenes/GameBoard.cs`
- Added `_lastSoundRound` field (int, tracks last round with sound)
- Updated `_Process()` to manage sound playback per round
- Reset `_lastSoundRound` in `EndTurn()` and `ClearSelection()`

**File:** `Core/GameObjects/Tile.cs`
- Removed per-tile sound calls from `MoveTile()` method
